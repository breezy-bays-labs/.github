# Contributing

Thanks for contributing to a Breezy Bays Labs project.

This file is the default contributing guide for repositories in the
`breezy-bays-labs` organization.

If a repository has its own `CONTRIBUTING.md`, that repository-specific guide
takes precedence over this default.

## Before You Start

- Read the repository README and any repo-specific architecture or workflow
  docs first.
- Follow the repository's issue templates and pull request template.
- Follow the organization's [Code of Conduct](./CODE_OF_CONDUCT.md).

## General Expectations

- Keep changes scoped and well explained.
- Include validation steps for the change.
- Update docs when behavior, APIs, configuration, or workflows change.
- Surface real risks and tradeoffs in the PR instead of hiding them.

## Pull Requests

Use the repository's pull request template and include:

- What problem changed
- What changed and what did not
- Commands run for validation
- Any user-facing, security, or migration impact

## Security

For vulnerabilities, do not use public issues. Follow the instructions in
[SECURITY.md](./SECURITY.md).

## Questions

If the repository has Discussions enabled, use that for usage and design
questions. Otherwise open an issue in the relevant repository.
